Links wont decrypt?

Important: Links encrypted before July 7 2026 won't be decryptable on the site.

Because the encryption key derivation incorporates a versioned domain string (currently valkcryption.com|msg|v2), links created with previous versions of the code (pre-July 7 2026) cannot be decrypted on the current hosted site at valkcryption.com.

To decrypt such links, you'll have to locally host the previous version yourself.

The previous version uses valkcryption.it|msg|v1 in the crypto code.

How to get & run the old version

Using git (recommended):

git clone https://github.com/00011110/valkcryption.git
cd valkcryption
git checkout v1

Or download a ZIP directly:

GitHub Releases - v1 (source zip under Assets)

or direct: https://github.com/00011110/valkcryption/archive/v1.zip

Then start it locally:

cp deploy/valkcryption.env.example .env
# Edit .env and set your own BASE_URL (e.g. http://127.0.0.1:8443)
node server.js

The v1 tag / commit uses valkcryption.com|msg|v1 and can decrypt links created before the July 7 2026 update.

Why did this happen?

The domain separator is mixed into the key derivation for message encryption (X25519 + HKDF + AES-GCM). This binds the ciphertext to a specific deployment/version and prevents certain cross-site attacks or config drift issues. When we update it, old ciphertexts become unreadable without the matching old code.

Self-hosting is fully supported — see DEPLOY.md and the open source repo. No accounts or data on our servers.
